# EDA001: 初期探索 - 全データセットの基本統計とテキスト特性分析

## 分析の目的

コンペ開始にあたり、全データファイルの基本統計を把握し、翻字・翻訳テキストの特性を理解する。
Train/Test間のギャップと、補足データの活用可能性を評価する。

## 分析概要

- 全10ファイルの基本情報（サイズ、行数、カラム）
- train.csv のテキスト長分布・特殊文字・記号分析
- test.csv との粒度差・語彙カバレッジ分析
- 補足データ（published_texts, publications, lexicon等）の概要

## 発見事項

### 1. train.csv の特性

- **1,561件**（欠損なし、重複oare_idなし）
- **ドキュメントレベル**のアラインメント（testは文レベル）

#### テキスト長

|------|-----------|-----------|
| 平均 | 56.2 | 90.0 |
| 中央値 | 49 | 68 |
| 最小 | 3 | 1 |
| 最大 | 150 | 739 |

- **翻訳/翻字 語数比**: 中央値 1.48（アッカド語1語 ≈ 英語1.5語に相当）

### 2. test.csv（ダミー）の特性

- ダミーは4文のみ、1ドキュメント由来
- 本番は**約4,000文、約400ドキュメント**
- **文レベル**のアラインメント（trainのドキュメントレベルとは異なる）
- test の `line_start` / `line_end` はint型（ダミーでは`'`付きstr値は未出現）

### 3. 特殊文字・記号分析

#### 限定詞（Determinatives）
- `{d}`（神）: 482回、`{ki}`（地名）: 383回
- 限定詞は2種類のみ出現 → trainデータは既に前処理済みか限定詞の出現が限定的

#### シュメール語ロゴグラム（ALL CAPS）Top 5
1. BABBAR: 3,390回（KÙ.BABBAR = 銀の一部）
2. DUMU: 1,901回（「息子」）
3. IGI: 1,128回
4. URUDU: 578回（「銅」）
5. AN.NA: 490回（「錫」）

#### 破損マーカー
- `<gap>`: 3,224回（高頻度）
- その他のマーカー（[x], ..., [… …], <big_gap>）: 0回
- → trainデータは既に `<gap>` に統一済み

#### Ḫ/ḫ 問題（重要）
- **train**: 84.1%（1,313/1,561）のテキストにḪ/ḫが含まれる
- **test**: 0%（ダミー4件中0件）→ テストは H/h のみ
- **対策**: 学習データの `Ḫ/ḫ` → `H/h` への正規化が**必須**

#### 翻訳テキスト
- 括弧 `()` を含むテキスト: 35.8%
- `...`, `[...]` 等の省略マーカー: 0%（既に除去済みか `<gap>` に変換済み）
- 翻訳にも `<gap>` が出現（翻字・翻訳両方に破損箇所あり）

### 4. 補足データ分析

#### published_texts.csv（8,000テキストの翻字 + メタデータ）
- 7,991件（うちtrain含有: 1,561件 → **train外: 6,392件**）
- ジャンル: unknown(4,046), letter(2,261), debt note(527), ...
- noteあり: 5,107件（64%）→ 翻訳のヒントになる可能性
- AICC_translation（機械翻訳）あり: 7,702件（97%）→ 品質低いが参考に

#### publications.csv（学術論文OCR）
- 240,510ページ、952件のPDF
- アッカド語含有ページ: 31,286ページ（12.9%）
- → **追加学習データの最大の源泉**

#### OA_Lexicon_eBL.csv（辞書）
- 39,332エントリ: word(25,574), PN=人名(13,424), GN=地名(334)
- → 人名・地名の翻訳精度向上に活用

#### Sentences_Oare_FirstWord_LinNum.csv（文アラインメント補助）
- 9,782エントリ
- **trainの文レベル分割に不可欠**（trainはdoc-level、testはsent-levelのため）
- `translation` カラムに文レベルの翻訳あり → 直接使える可能性

#### bibliography.csv
- 出版年: 1882〜2025年（中央値2000年）

### 5. Train-Test ギャップ

- Train語彙: 11,583語
- Test語彙（ダミー）: 59語（カバレッジ: 52.5%）
- Test-only語の例: `a-lim(ki)`, `kà-ni-ia`, `mup-pì-ni` 等
- → ダミーが4文のみなので統計的意味は薄いが、**新語（OOV）は確実に発生する**

## 精度向上の仮説

### 堅実案
1. **Ḫ/ḫ → H/h 正規化**: train 84%に含まれるため必須の前処理
2. **Sentences_Oare_FirstWord_LinNum.csv を活用した文レベルアラインメント構築**: trainのdoc→sent分割
3. **OA_Lexicon による人名・地名辞書の活用**: 固有名詞翻訳の改善
4. **publications.csv からの追加ペア抽出**: 学習データ量の増大

### 爆発案
1. **AICC機械翻訳（7,700件）を弱い教師信号として使用**: ノイズ付きだがdata augmentationとして活用、品質フィルタリングと組み合わせ
2. **シュメール語ロゴグラム（ALL CAPS）の事前翻訳テーブル作成**: ルールベースで確定的に翻訳できる部分を先に処理し、NMTの負荷を軽減
3. **eBL辞書からの形態素レベルの知識注入**: アッカド語の形態素分析をサブワードトークナイザーに反映
