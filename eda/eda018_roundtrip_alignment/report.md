# eda018: Round-trip翻訳によるAkkadian文レベルアライメントの可能性調査

## 分析の目的

exp016(ByT5-base)の逆翻訳(Eng→Akk)品質を確認し、英語の文分割→逆翻訳でAkkadian側のsentence boundaryを推定できるかを検証する。

## 分析概要

- **対象**: train.csvの複数文ドキュメント (1,191/1,561件 = 76.3%)
- **サンプル数**: 50件 (ランダム抽出)
- **手順**:
  1. 英語翻訳を文分割 (ピリオド区切り)
  2. 各英語文をEng→Akkに逆翻訳 (beam4)
  3. 逆翻訳Akkと元Akkの類似度をchrF++で評価
  4. 文レベルアライメントの可能性を対角一致率で評価

## 発見事項

### 1. Round-trip翻訳品質は高い

| 指標 | 値 |
|------|-----|
| chrF++ (逆翻訳Akk vs 元Akk) mean | **74.60** |
| chrF++ median | **74.94** |
| バイト比 (roundtrip/orig) | 1.04 (ほぼ同じ長さ) |
| chrF++ >= 70 | 56% (28/50) |
| chrF++ >= 60 | 88% (44/50) |
| chrF++ >= 50 | 98% (49/50) |

→ 逆翻訳の品質は**かなり高い**。元のAkkadianを概ね再現できている。

### 2. 文レベルアライメントの対角一致率

均等分割チャンクと逆翻訳文のchrF++マッチングで順序保存を評価:

| 文数 | 対角一致率 | サンプル数 |
|------|-----------|-----------|
| 2文 | **100.0%** | 12 |
| 3文 | **73.3%** | 30 |
| 4文 | **68.8%** | 32 |
| 5文 | 51.1% | 45 |
| 6文 | **70.0%** | 30 |
| 7文 | 28.6% | 14 |
| 8文以上 | 低い | - |
| **全体** | **54.7%** | 254 |

→ 2-4文のドキュメントでは順序保存率が高い（68-100%）。5文以上では不安定。

### 3. 上位のround-trip例

**chrF++=100.0 (完全再現)**:
```
AKK_ORIG:      10 ma-na KÙ.BABBAR ṣa-ru-pá-am tù-ur-a-a DUMU ú-zu-a...
AKK_ROUNDTRIP: 10 ma-na KÙ.BABBAR ṣa-ru-pá-am tù-ur-a-a DUMU ú-zu-a...
```
→ 定型的な商取引文書では完全再現が可能

### 4. 実用性の評価

**使える場面 (2-4文、chrF++>70)**:
- 定型的な文書（契約、証人リスト、借用書）
- 短いドキュメント
- 76.3%のドキュメントが複数文 → 対象は十分

**使えない場面**:
- 5文以上の長い文書 → アライメント精度が急落
- `<gap>`を含む断片的テキスト
- 非定型的な手紙文

## 結論

- **Round-trip翻訳品質自体は高い** (chrF++ mean=74.6)
- **2-4文のドキュメントでは文レベルアライメントが有望** (対角一致率69-100%)
- ただし、これはあくまで均等分割ベースの粗い評価。実際のアライメントにはより精密な手法が必要
- **文レベルデータ拡張としてのポテンシャルはある**が、5文以上のドキュメントでは品質が不安定
- 追加データの品質が課題であることを考えると、round-trip生成データもドメインミスマッチの同様の問題を抱える可能性あり
