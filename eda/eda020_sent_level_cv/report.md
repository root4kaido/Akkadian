# eda020: Sentence-level CV vs Doc-level CV（切り詰めなし）

## 目的
- 従来のCV（200B截断+1文抽出）がLBと乖離する原因を特定
- sentence_aligned.csvを使い、6文以下のvalドキュメントを文分割して文単位CV(sent-CV)を計測
- doc-CV（切り詰めなし、全文翻訳）も計測し、モデルの多角的評価を確立

## 分析概要
- exp015/016/017/018/019の5モデルで統一評価
- val: 157 docs（exp016と同一split）
- sent-CV: 57 docs → 189 sentences（6文以下でsentence_aligned.csvにアライメントあり）
- doc-CV: 157 docs全体（入力も参照も切り詰めなし）

## 発見事項

### 1. 旧CVの膨張原因
- 200B截断 + 最初の文のみ = 文脈ありの短文翻訳。テスト条件（文脈なし文レベル）と大きく異なる
- 旧CV/LB比=1.55-1.66と不安定

### 2. sent-CVがLBに最も近い
- sent-CV/LB比=1.23-1.39（旧CVより安定）
- テスト条件に近い（文脈なし、文レベル入力）
- ただしval分布とtest分布の差は残る（exp015が高く出る問題）

### 3. doc-CVは繰り返し耐性を測る
- 全モデルでrep=53-60%と壊滅的
- doc-CVがLBより低い場合も（exp016: 25.41 < LB 29.5）
- testは短文なので繰り返しが起きにくい

### 4. 推奨評価方法
- **sent-CV + doc-CV の両方で評価する**のが最善
- sent-CV: LBとの相関が高く、モデルの翻訳品質を測る
- doc-CV: 繰り返し耐性・汎化力を測る
- 両方で改善する施策がLBでも効く可能性が高い

### 5. exp019がsent-CVで全モデル最高
- sent-CV=39.50（exp016比+2.55pt）
- 文分割データで学習した効果が文レベル評価で顕著に出ている
- LB提出で確認すべき
