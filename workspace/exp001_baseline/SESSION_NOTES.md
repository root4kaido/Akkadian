## 知見

1. **文アライメントが完全に無効** — trainデータのtransliterationに改行(`\n`)が一切含まれず、`src.split('\n')`が常に1行返す → アライメント条件が絶対にTrueにならない（0/1561行）。さらに`normalize_text()`の`\s+→' '`変換が仮に存在した改行も潰す
2. **`padding="max_length"`が最大のスコア低下要因**（推定-5~8pt）— 入力・ラベルの両方を512トークンに固定パディング。ByT5はバイトレベルなので平均入力長50-200バイトに対して大量の無駄パディング。Starterは動的パディング（DataCollatorに委任）
3. **`normalize_text()`による情報損失**（推定-2~4pt）— `Ḫ→H`変換はアッカド語で意味のある音素区別を潰す。Starterは一切正規化せず生テキストで学習
4. **スコアはepoch 16-18で収束** — lossは微減だがメトリクスは頭打ち

### Starterとの主要差分

| 項目 | Starter | exp001 | 影響 |
|------|---------|--------|------|
| padding | 動的(DataCollator) | **max_length=512固定** | **最大要因** |
| 前処理正規化 | なし | Ḫ→H, 下付き数字, NFC | 情報損失 |
| 英語翻訳の正規化 | なし | normalize_text適用 | 不要・有害 |
| 後処理 | 空→broken textのみ | 多数のルール | CV計算には無関係 |
| その他HP | 同一 | 同一 | - |

## 次のステップ (exp002で修正)

1. **padding="max_length" を削除** → DataCollatorによる動的パディングに切り替え（最優先）
2. **normalize_text() を無効化 or 最小化** → Starterと同様に生テキストで学習
3. **英語翻訳への正規化を停止**
4. 上記修正でStarter相当の~30を再現してから次の改善へ
